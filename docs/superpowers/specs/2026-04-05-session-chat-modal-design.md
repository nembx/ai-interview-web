# 会话聊天窗口弹窗化设计

## 目标

将当前 RAG 会话工作台底部的嵌入式聊天区域改为点击会话列表项后打开的居中大弹窗，解决以下问题：

- 聊天区域固定在页面底部，视觉优先级过低
- 当前宽度偏小，长回答阅读体验差
- 流式生成过程中消息区域持续增高，导致界面闪动

## 范围

本次改动仅调整前端会话聊天的展示和交互，不改动后端接口、消息数据结构和会话管理能力。

保留内容：

- 会话创建、刷新、归档、删除
- 会话标题编辑和知识源管理
- 现有 SSE 流式问答逻辑

调整内容：

- 点击会话列表项时直接打开聊天弹窗
- 工作台中移除底部嵌入式消息区
- 聊天改为大尺寸模态窗口

## 交互设计

### 打开方式

- 用户点击会话列表中的任意会话项
- 前端先加载该会话详情
- 成功后弹出聊天窗口并定位到当前会话底部消息

### 弹窗布局

- 桌面端使用居中大弹窗，宽度约 `min(1100px, 88vw)`，高度约 `82vh`
- 移动端降级为接近全屏
- 弹窗内部采用三段式布局：
  - 顶部：会话标题、状态、关闭操作
  - 中部：固定高度消息滚动区
  - 底部：提问输入框和发送按钮

### 页面主体调整

- 会话工作台保留标题编辑、状态切换、时间信息、知识源选择
- 原工作台右侧聊天区域删除，避免页面底部出现重复会话视图

## 流式消息与防闪动策略

### 固定滚动容器

- 消息区使用固定高度的弹性布局容器
- 新消息只更新内部滚动内容，不再撑高外层布局

### 流式输出渲染

- assistant 生成期间仅维护一条正在更新的消息气泡
- 随流内容增量更新该消息内容，避免多节点反复插入造成重排

### 自动滚动规则

- 打开弹窗时默认滚到底部
- 仅当用户当前接近底部时，流式更新才自动跟随到底部
- 用户手动上滑查看历史消息后，不强制抢占滚动位置

### 文本稳定性

- 消息正文使用更稳定的换行样式，如 `whitespace-pre-wrap`
- 统一消息气泡的内边距和行高，降低流式生成时的视觉跳动

## 实现方式

- 在现有状态基础上新增聊天弹窗开关和当前打开会话状态
- 将聊天 UI 从 `App.tsx` 中抽出为独立组件，降低主文件复杂度
- 继续复用现有的会话详情、发送问题、流式更新逻辑

## 错误与加载状态

- 会话详情加载中时，弹窗消息区显示轻量 loading 状态
- 会话已归档时，输入区保持禁用并展示现有提示文案
- 会话发送中时，发送按钮保持禁用，避免重复提交

## 验证点

- 点击会话列表项可稳定打开对应聊天弹窗
- 弹窗在桌面和移动尺寸下都可正常阅读和输入
- 长回答生成时弹窗尺寸保持稳定，不因内容增长闪动
- 用户位于底部时自动跟随；离开底部后不会被强制拉回
- 原有创建、编辑、归档、删除、发送消息流程不回归
